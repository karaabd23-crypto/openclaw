# Archive

## What belongs in archive

- Past session summaries that are no longer “active context”.
- Closed decisions and resolved questions with date/source.
- Historical snapshots of priorities after they are superseded.

## How session summaries should be stored

- One markdown file per session.
- File naming pattern:
  - `YYYY-MM-DD_<short-title>.md`
- Keep summaries factual and concise.
- Include unresolved items explicitly.
- Do not store secrets, tokens, or private credentials.
